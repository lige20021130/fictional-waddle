========================================
JSON输出路径配置使用说明
========================================

新功能说明
---------
现在你可以为两种JSON文件分别指定输出路径:

1. 中间任务 JSON (后处理层输出)
   - 文件格式: {pdf文件名}_mid_task.json
   - 包含: LLM文本提取任务 + VLM图像提取任务
   - 默认位置: PDF文件同级目录
   - 自定义: 在"后处理"区域点击"选择目录"

2. 提取结果 JSON (大模型输出)
   - 文件格式: {pdf文件名}_mid_task_extracted.json
   - 包含: 最终的字段提取结果
   - 默认位置: ./extraction_results 目录
   - 自定义: 在"大模型提取"区域点击"选择目录"


使用场景
--------
场景1: 集中管理所有中间文件
  - 设置中间JSON输出为: D:\project\mid_tasks
  - 所有PDF的中间任务JSON都会保存到这里

场景2: 分别管理不同项目的结果
  - 项目A的提取结果: D:\project_A\results
  - 项目B的提取结果: D:\project_B\results

场景3: 使用默认路径(留空即可)
  - 中间JSON: 保存在PDF同级目录
  - 提取结果: 保存在 ./extraction_results


操作步骤
--------
1. 启动GUI后,找到"后处理"区域
2. 点击"中间JSON输出"旁的"选择目录"按钮
3. 选择你想要的保存位置(或留空使用默认)
4. 找到"大模型提取"区域
5. 点击"提取结果输出"旁的"选择目录"按钮
6. 选择你想要的保存位置(或留空使用默认)
7. 正常进行PDF处理和大模型提取


注意事项
--------
✓ 路径为空时使用默认路径
✓ 目录不存在时会自动创建
✓ 每次处理会在日志中显示实际保存路径
✓ 可以在处理过程中查看日志确认输出位置


日志示例
--------
[配置] 中间JSON输出目录: D:\data\mid_tasks
[配置] 提取结果输出目录: D:\data\results
[预处理] ✓ 预处理完成，中间任务JSON: D:\data\mid_tasks\paper_mid_task.json
[提取] 使用自定义输出目录: D:\data\results
[提取] 结果保存至: D:\data\results\paper_mid_task_extracted.json
