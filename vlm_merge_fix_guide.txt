========================================
一、VLM提取器TypeError修复
========================================

问题描述:
---------
错误: TypeError: list indices must be integers or slices, not str
位置: vlm_extractor.py 第82行
原因: 在asyncio.gather中直接修改返回值导致数据结构混淆

问题根源:
---------
原始代码:
  async def bounded(task):
      result = await self._extract_from_image(...)
      result['_source'] = task  # ❌ 在协程内部修改
      return result
  
  results = await asyncio.gather(*tasks)
  # results是List[Dict],但每个Dict可能没有'_source'键

修复方案:
---------
使用元组返回,在后处理时添加_source:

  async def bounded(task):
      result = await self._extract_from_image(...)
      return (result, task)  # ✓ 返回元组
  
  raw_results = await asyncio.gather(*tasks)
  # raw_results是List[Tuple[Dict, Dict]]
  
  final_results = []
  for res, src_task in raw_results:
      res['_source'] = src_task  # ✓ 安全添加
      final_results.append(res)

测试验证:
---------
✓ 处理了3个图像任务
✓ 每个结果都正确包含_source字段
✓ _source包含image_path和caption信息


========================================
二、智能合并高价值文本块
========================================

设计理念:
---------
将连续的高价值信息文本块合并,减少LLM调用次数,
同时保持上下文连贯性。

高价值信息定义:
--------------
1. 动力学参数: Km, Vmax, Michaelis-Menten
2. 催化活性: catalytic, activity, peroxidase, oxidase
3. 表征手段: TEM, XRD, XPS
4. 材料合成: synthesis, nanozyme, nanomaterial
5. 底物相关: substrate, TMB, H2O2
6. 预提取的化学式: 如Fe3O4, CoFe2O4等

合并策略:
---------
1. 遍历所有文本块
2. 检查每个块是否包含高价值关键词
3. 如果连续多个块都是高价值,合并成一个大块
4. 低价值块保持独立(也可选择过滤)
5. 保留章节和页码信息

示例:
-----
原始7个块:
  块1: "This is general introduction text." [普通]
  块2: "We synthesized Fe3O4 nanoparticles." [高价值: synthesized]
  块3: "The Km value was 0.5 mM." [高价值: Km]
  块4: "Vmax = 2.5×10^-3 M s-1." [高价值: Vmax]
  块5: "Some other results." [普通]
  块6: "TEM images showed 5nm particles." [高价值: TEM]
  块7: "Conclusion of the paper." [普通]

合并后5个块:
  块1: "This is general introduction text." [普通]
  块2: "We synthesized... Km value... Vmax =..." [高价值合并: 块2+3+4]
  块3: "Some other results." [普通]
  块4: "TEM images showed 5nm particles." [高价值]
  块5: "Conclusion of the paper." [普通]

减少比例: 28.6% (7→5个块)


========================================
三、使用方式
========================================

默认使用(推荐):
--------------
# 自动使用合并后的高价值块
pre.to_mid_json("output.json")
# 等同于
pre.to_mid_json("output.json", use_merged=True)

使用原始块:
-----------
# 不使用合并,使用所有原始块
pre.to_mid_json("output.json", use_merged=False)

使用RAG筛选:
-----------
# RAG筛选优先级高于合并
pre.to_mid_json("output.json", use_rag=True, top_k=10)

组合使用:
---------
# 先合并,再RAG筛选(适合超长文献)
pre.to_mid_json("output.json", use_merged=True, use_rag=True, top_k=10)

========================================
五、技术细节
========================================

新增方法:
---------
_merge_high_value_chunks(chunks: List[TextChunk]) -> List[TextChunk]
  功能: 合并连续的高价值文本块
  策略: 
    - 识别高价值块(关键词匹配)
    - 连续高价值块合并
    - 保留页码和章节信息
  返回: 合并后的TextChunk列表

修改方法:
---------
process():
  新增: 调用_merge_high_value_chunks()
  存储: self.merged_chunks

to_mid_json(save_path, use_rag, top_k, use_merged=True):
  新增参数: use_merged (默认True)
  优先级: use_rag > use_merged > 原始块

========================================
七、注意事项
========================================

⚠ 合并可能丢失段落边界
  - 合并后的块包含多个原始块的内容
  - 段落之间用"\n\n"分隔
  - LLM仍能识别段落结构

⚠ 高价值关键词可能不全面
  - 当前覆盖常见关键词
  - 如发现遗漏,可在HIGH_VALUE_TERMS中添加

⚠ 合并不等于过滤
  - 低价值块仍然保留
  - 只是高价值块被合并
  - 如需过滤,可修改_merge_high_value_chunks逻辑

⚠ use_rag优先级更高
  - 如果同时设置use_rag=True和use_merged=True
  - 系统会先合并,再RAG筛选
  - 这可能导致过度筛选,建议二选一
